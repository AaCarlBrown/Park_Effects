# Mesoball: Park Effects at Baseball's Middle Scale

Reproduction code for the paper *Mesoball: Park Effects at Baseball's
Middle Scale* (submitted to the SABR Baseball Research Journal). Every
number in the paper is produced by the staged pipeline in this repository.

The method rates every batter and pitcher against the actual opposition
they faced via a two-way fixed-effects model over 15.4 million plate
appearances (1910-2025), then reads park effects out of the residuals,
season by season, using only information available at the time. The paper
demonstrates what that resolution buys: a statistically dated catalogue of
19 sharp park breaks, a prediction contest against rebuilt
FanGraphs/Baseball-Reference factors, park effects split by handedness and
hitter type, career adjustment ledgers, fielding effects, and a
pre-registered test of lineup-to-park fit.

## Pipeline

Stages run in order from this directory (Windows: `py u_XX_name.py`;
elsewhere: `python u_XX_name.py`). Each stage checks pre-registered gates
in `Expectations/` and refuses to pass silently on surprises. Smoke tests
(`u_XXs_smoke.py`) run on synthetic data and take seconds.

| Stage | Script | What it does |
|---|---|---|
| 0 | `u_00_setup.py` | Verify inputs against expectations (row counts, columns, seasons) |
| 0b | `u_00b_get_inputs.py` | Download Retrosheet game logs, park codes, NOAA ISD-lite wind |
| 1 | `u_01_derive.py` | Merge game logs onto the PA panel; derive batter side, park, season fields |
| 1b | `u_01b_diagnose.py` | Coverage diagnostics |
| 2 | `u_02_skill_model.py` | Two-way fixed-effects wOBA model (pyfixest); ages, anchoring |
| 2b | `u_02b_exante.py` | 115 expanding-window refits for the real-time series (hours; checkpointed) |
| 3 | `u_03_park_series.py` | Park-by-season effects from residuals, league-relative within connectivity components |
| 3b | `u_03b_breaks.py` | Changepoint screen (recursive binary segmentation), regimes, shrinkage |
| 3c | `u_03c_break_postmortem.py` | Break catalogue vs the paper's claims, both directions |
| 3f | `u_03f_figures.py` | Figures 1-2 (ex-ante shrunk real-time line vs conventional dots) |
| 4 | `u_04_conventional.py` | Rebuild BR-style and FG-style factors from game logs |
| 4b | `u_04b_contest.py` | Prediction contest, two currencies, locked gates |
| 4c | `u_04c_woba_conversion.py` | Park run-conversion residual (wOBA-scale corrected) |
| 4d | `u_04d_seamheads.py` | External cross-check vs the Seamheads database (local copy required) |
| 5 | `u_05_wind.py` | Wrigley wind: 5,000+ day games matched to hourly airport records |
| 6 | `u_06_splits.py` | Handedness, hitter type, quality splits; hand-by-type interaction |
| 7 | `u_07_ledger.py` | Career ledgers, mesoball vs conventional adjustments |
| 8 | `u_08_fielding.py` | Park fielding effects (compositional putout shares) |
| 8b | `u_08b_park_alloc.py` | Pre-registered lineup-fit test (registered null) |
| 9 | `u_09_validate.py` | Consolidated validation report across all stages |

## Pre-registration

`Expectations/expectations_NN.json` files were written before each stage
ran. Where a first run failed its gates, the file's `revisions` block
records what was learned and what changed; failures are findings. Two
externally caught errors are documented in the relevant docstrings
(`u_04c`: wOBA-scale unit mix; `u_03f`: unapplied shrinkage).

## Inputs (not included)

Place these in `data/` (paths configurable in `config.py`; root overridable
via the `SABR_MESOBALL_ROOT` environment variable):

- `pa_panel.parquet` - one row per plate appearance, 1910-2025 (15,374,332
  rows; required columns `bat_id`, `pit_id`, `game_id`, `woba_value`).
  Built from Retrosheet event files; Stage 0 verifies it.
- `wOBA_weights.csv` - season wOBA weights and scale (FanGraphs guts table).
- `Teams.csv` - Lahman database teams table.
- `biofile0.csv` - Retrosheet biographical file.
- `pitching.csv` - Lahman pitching table.
- `plays.csv` - Retrosheet event-level plays (Stage 8 only; large; path via
  `SABR_PLAYS_CSV`).
- Seamheads Ballparks database CSVs (Stage 4d only): licensed for
  individual research use, so **not** redistributed here. Obtain from
  seamheads.com and point `SABR_SEAMHEADS_DIR` at the folder.

Stage 0b downloads Retrosheet game logs, the park-code crosswalk, and NOAA
ISD-lite wind files into `data/` automatically.

## Results

`results/` contains the small output files the paper's tables and claims
are drawn from (breaks catalogue, regimes, contest results, splits,
ledgers, conversion residuals, fielding effects, validation report).
Large intermediates (`pa_derived.parquet`, ex-ante checkpoints) are
regenerated by the pipeline. Seamheads-derived output is excluded per its
license.

## Requirements

Python 3.10+ with `pandas`, `numpy`, `pyarrow`, `pyfixest`, `matplotlib`
(see `requirements.txt`). The full rerun takes several hours, dominated by
Stage 2b's expanding-window refits.
